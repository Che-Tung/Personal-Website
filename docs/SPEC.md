# 個人網站規格文件

## 1. 全站共通功能（Global）

### 1.1 Layout 與導覽

#### 描述

全站需有一致的版型與導覽結構。

#### 功能需求

- 固定 Header，包含：
  - Logo / Name
  - 導覽連結（Home / Projects / Blog / About）
- Footer 顯示：
  - Copyright
  - Social links（GitHub / LinkedIn）

#### 驗收條件（Acceptance Criteria）

- 所有頁面都套用相同 layout
- 導覽可正確切換頁面

#### Out of Scope

- 動畫導覽列
- 深色模式切換（v2 再做）

---

## 2. 首頁（Home）

### 2.1 個人簡介區塊

#### 描述

展示個人基本資訊與定位。

#### 功能需求

- 顯示姓名
- 一段簡短自我介紹
- 一個 CTA（導向 Projects 或 Blog）

#### 驗收條件

- 首頁載入即可看到完整介紹
- CTA 點擊後導向正確頁面

---

## 3. 作品集頁（Projects）

### 3.1 作品列表

#### 描述

以卡片形式展示個人專案。

#### 功能需求

- 每個專案包含：
  - 專案名稱
  - 簡短描述
  - 使用技術
  - 專案連結（GitHub / Demo）
- 專案資料來源為本地檔案（JSON / MD）

#### 驗收條件

- 專案可正確渲染列表
- 點擊連結可開新分頁

#### Out of Scope

- 專案搜尋 / 篩選
- 動態後台管理

---

## 4. 文章列表頁（Blog）

### 4.1 文章列表

#### 描述

展示所有文章摘要，供使用者瀏覽。

#### 功能需求

- 顯示：
  - 標題
  - 發佈日期
  - 摘要
- 文章依日期排序（新到舊）
- 點擊文章導向詳細頁

#### 資料來源

`/content/blog/*.mdx`

#### 驗收條件

- 新增文章檔案後，列表自動更新
- 路由正確

---

## 5. 文章詳細頁（Blog Detail）

### 5.1 文章內容呈現

#### 描述

顯示單篇文章完整內容。

#### 功能需求

- 支援 Markdown / MDX
- 支援：
  - code block
  - 圖片
  - 標題層級（h1–h3）
- 顯示文章標題與日期

#### SEO

- 每篇文章有獨立 metadata
- URL 為 `/blog/[slug]`

#### 驗收條件

- 任一文章可直接透過 URL 存取
- 重新整理不會出錯（SSG）

#### Out of Scope

- 留言系統
- 按讚 / 分享功能

---

## 6. About 頁

### 6.1 個人背景說明

#### 描述

- 更完整的自我介紹
- 技能摘要
- 聯絡方式

#### 功能需求

- 靜態內容即可
- 可包含外部連結

---

## 7. 非功能需求（Non-functional Requirements）

### 7.1 SEO

- 所有頁面設定基本 metadata
- Open Graph for sharing

### 7.2 RWD

- 手機 / 平板 / 桌機可正常瀏覽

### 7.3 效能

- 靜態頁面優先使用 SSG
- 首屏快速渲染

---

## 8. 技術限制與決策

- 不使用資料庫（v1）
- 不實作後台管理（v1）
- 不使用第三方 CMS（v1）
- 使用 Git 作為內容管理工具
